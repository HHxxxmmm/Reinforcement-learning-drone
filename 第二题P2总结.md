## 1. P2 任务设计

| 项 | P1 | P2 |
|----|----|----|
| 锁定 | roll + **yaw** | **仅 roll** |
| 学习 | pitch + 油门 | pitch + 油门 + **yaw** |
| 敌机 y | 0 | **0**（P3 再随机） |
| yaw 奖励 | `yaw_misalign_weight: 0` | **8.0** |

**子目标**：继承 P1 的爬升–俯冲–击杀节奏，在 y=0 上学会 **极小 yaw 修正**，勿大甩头。

---

## 2. 两阶段路线：P2 初版 → P2 v2

### 2.1 P2 初版（`max_yaw: 0.15`）

| 项 | 设定 |
|----|------|
| 配置 | `envs.junior.phase2.yaml` / `algs.junior.phase2.yaml` |
| 起点 | P1@1k + **`reset_yaw_head_on_load: true`** |
| PPO | lr=5e-5，n_epochs=3，save_freq=1k |

**eval（y=0，3 局）**：`ppo_junior_p2_1000_steps.zip` → **3/3 击杀**，169 步，final_yaw **−0.29°**。

结论：reset yaw 有效，P1 pitch+油门策略可平滑迁移。

### 2.2 P2 v2（解除 `max_yaw`）

| 项 | 设定 |
|----|------|
| 配置 | `envs.junior.phase2.v2.yaml` / `algs.junior.phase2.v2.yaml` |
| 起点 | P2@1k（**不 reset yaw**） |
| 动作 | 删除 `max_yaw`，舵量由 `action_scale=0.50` 决定（±0.5 rad 上限） |
| 目录 | `logs/junior_q2_phase2_v2/`、`model/junior_q2_phase2_v2/` |

**动机**：y=0 上 1k 已稳定，放宽 yaw 上限以备 P3 侧向靶；同时验证续训是否再次「早期峰值、后期坍缩」。

**训练现象**：续训至 ~12k 步时 UE 断连；对 **1k–12k 共 12 个 ckpt** 做批量 eval（`eval_junior_phase2_v2.py`，每 ckpt 3 局）。

---

## 3. 关键工程：yaw 头 reset

与 Q1 P1→P2 同类问题（见 `第一题总结.md` §5.2、§6）：

| | Q1 P1@30k | Junior P1@1k（实测） |
|--|-----------|----------------------|
| 锁 yaw 时 action[3] 漂移 | **−0.11 ~ −0.20**（左偏） | bias **+0.058**，det **+0.315** |
| 解锁后不 reset | 持续左拐，0 击杀 | **打满 max_yaw≈0.15 rad 右转** |
| 处理 | P2 v2 + reset_yaw | **`reset_yaw_head_on_load: true`** |

reset 后：action[3]≈0，throttle/pitch bias **保留**（0.76 / 0.24）。

## 4. Eval 结果（room 20411 / port 1004）

环境：`envs.junior.phase2.v2.eval.yaml`，deterministic × 3 局 / ckpt。  
汇总 CSV：`logs/junior_q2_phase2_v2/eval/phase2_v2_summary.csv`

| ckpt | kills | steps | pitch° | yaw° | dy_m | dz_m | hp | 备注 |
|------|-------|-------|--------|------|------|------|-----|------|
| **1000** | **3/3** | **169** | **+2.9** | **−1.33** | **−4.0** | **+0.4** | **0.00** | **定稿：轨迹最紧、reward 最高** |
| 2000 | 3/3 | 170 | +6.1 | −3.52 | −10.9 | +3.6 | 0.00 | 仍可杀，侧向/高度偏移变大 |
| 3000 | 0/3 | 249 | +27.9 | −7.78 | −37.0 | +86.0 | 0.27 | pitch 饱和，overshoot 截停 |
| 4000 | 0/3 | 249 | +26.8 | −7.44 | −35.8 | +85.3 | 0.20 | 同上 |
| 5000 | 3/3 | 169 | +8.1 | −2.35 | −7.4 | +11.1 | 0.00 | 偶发恢复击杀，dz 已偏大 |
| 6000 | 3/3 | 169 | +6.2 | −1.41 | −4.4 | +9.7 | 0.00 | 可杀但不如 1k |
| 7000 | 0/3 | 244 | +27.1 | −1.34 | −7.4 | +100.3 | 0.31 | 爬升过高，未杀 |
| 8000–12000 | 0/3 | 238–241 | **+30.0** | ±0.3~+3.2 | 变化 | **+122~+184** | 0.46–0.81 | pitch 顶满 max_pitch，飞过靶 |

### 4.1 范式（与 P1/Q1 一致）

> **续训前 1k 步最好；继续更新 → pitch 贴顶 +30°、dz 飙升、overshoot 截停、击杀率归零。**

1k 定稿局末典型量（v2 eval）：

```
steps=169  pitch=+2.90°  yaw=-1.33°  dy=-4.0m  dz=+0.4m  enemy_hp=0.000  reward≈3996
```

策略模式：与 P1 相同 **爬升→俯冲接敌→击杀**，yaw 仅微调。

## 5. 定稿设计与推荐 ckpt

### 5.1 动作 / 奖励（P2 v2 定稿 env）

文件：`config/envs.junior.phase2.v2.yaml`

```yaml
enemy_y_range: [0, 0]
yaw_misalign_weight: 8.0
# … altitude / speed / attack 同 P1 v8-A-tuned …
action:
  lock_roll: true
  lock_pitch: false
  max_pitch: 0.35
  lock_yaw: false
  # 无 max_yaw
  max_throttle: 0.46
  action_scale: 0.50
```

### 5.2 算法（P2 初版 hand-off）

```yaml
load_path: ppo_junior_p1_1000_steps.zip
reset_yaw_head_on_load: true
learning_rate: 5.0e-5
n_epochs: 3
save_freq: 1000
```

### 5.3 推荐 checkpoint

| 路径 | 说明 |
|------|------|
| **`model/junior_q2_phase2_v2/ppo_junior_p2_v2_1000_steps.zip`** | **P2 定稿 → 进 P3** |
| `model/junior_q2_phase2/ppo_junior_p2_1000_steps.zip` | P2 初版（max_yaw=0.15），亦 3/3，作对照 |
| `model/junior_q2_phase1/ppo_junior_p1_1000_steps.zip` | P1 定稿 |

## 6. 踩坑清单

| 现象 | 原因 | 处理 |
|------|------|------|
| P2 初版右拐 | P1 锁 yaw → action[3] 正漂移 | **reset_yaw_head_on_load** |
| v2 续训 3k+ 不杀 | pitch 学满 +30°，爬升过高飞过靶 | **1k 早停**；P3 保守 PPO |
| dz 从 0.4m 涨到 180m+ | 高度–速度–pitch 耦合失控 | eval 看 **dz_m** 比 reward 更敏感 |
| 5000/6000 仍 3/3 但非最优 | 局部恢复，dz/yaw 已劣化 | 以 **1k** 为准，勿贪步数 |
| TensorBoard 续训 reward 仍高 | 稀疏击杀 + value 漂移 | **批量 ckpt eval** 定稿 |

## 8. 常用命令

## 9. P3 展望

| 项 | 计划 |
|----|------|
| 起点 | **`ppo_junior_p2_v2_1000_steps.zip`** |
| 泛化 | `enemy_y_range: [-5, 5]`，`enemy_y_positive_prob: 0.70` |
| 动作 | 保持 pitch+yaw+油门，无 max_yaw |
| PPO | 保守续训：lr≈3e-5，n_epochs=3，**save_freq=1k**，短步 + y 扫点 eval |
| 过关 | y∈[-5,5] 扫点 eval，参考 Q1 11/11 或 Junior 可行 K 值 |

配置：`config/envs.junior.phase3.yaml`、`config/algs.junior.phase3.yaml`

## 10. 一句话

**Junior P2 自 P1@1k 解锁 yaw 并 reset 漂移维；P2 v2 解除 max_yaw 后全量 eval 证实 1k ckpt 最优（3/3、pitch≈+3°、dz≈+0.4m），续训 3k+ pitch 饱和爬飞；定稿 `ppo_junior_p2_v2_1000_steps` 进入 P3 y 随机泛化。**
