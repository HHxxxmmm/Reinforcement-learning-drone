## 三、关键联调结论

### 3.1 房间类型与脚本选择

| UE 房间类型 | 使用脚本 | 连接方式 |
|-------------|----------|----------|
| Simple vs **固定靶机** | `smoke_test.py` / `main.py` | 只连 **单位0 控制端口** |
| Simple vs **Simple**（双智能体） | `dual_smoke_test.py` | **同时**连单位0、单位1 端口 |

Simple vs Simple 只连一端会导致 initial 后 `TimeoutError`（平台等待双端都连上）。

### 3.2 UE 建房参数建议

| UE 设置 | 平台字段 | 建议（`total_timesteps=20000` 短训） |
|---------|----------|----------------------------------------|
| **比赛轮数** | `maxEpisodes` | **25～30**（按总步数÷每局步数估算，略留余量） |
| **最大回合数** | `maxStepsPerEpisode` | **1000**（与实测局长一致）或 10000（上限，不一定会跑满） |

**注意：** 比赛轮数过小（如 1）时，第一局结束后房间回收，训练第二次 `reset` 会 `ConnectionRefused`。

### 3.3 Simple 场景 HP 尺度

- 接口文档写 0～1000「点」，**Simple 实测观测为 0～1 浮点**（`1.0`=满血）。
- 打印 HP 时需保留小数（`%.3f`），用 `%.0f` 会把 `0.46` 显示成 `0`。

### 3.4 固定靶机观测特点

- `enemy_hp` 有更新，但 `enemy_pos` 常为 `(0,0,0)`；特征与奖励设计时需与队友确认是否用相对坐标或其它字段。

### 3.5 PPO 训练步数取整

`total_timesteps=20000`、`n_steps=512` 时，实际环境步数为 `ceil(20000/512)×512 = 20480`，属 Stable-Baselines3 正常行为。

## 四、使用说明

### 4.1 环境准备

```powershell
cd Python
.\venv\Scripts\Activate.ps1
python -c "from envs.train_env import TrainEnv"   # 验收 import
```

### 4.2 配置

**`config/envs.yaml`**（每次新建 UE 房间后必改）：

```yaml
host: 10.119.14.141
port: <单位0控制端口>
unit_id: 0
room_id: <加入房间 uid>
train_mode: online
```

**`config/algs.yaml`**：训练步数、PPO 超参、checkpoint 频率等。

### 4.3 阶段 1 冒烟

```powershell
# UE 已进入对战画面
python scripts/smoke_test.py --log-interval 200 --check-env
```

### 4.4 阶段 2 训练

```powershell
python main.py
tensorboard --logdir ./logs/single_training/
# 浏览器打开 http://localhost:6006
```

### 4.5 产出物路径

| 路径 | 内容 |
|------|------|
| `./model/ppo_simple_*_steps.zip` | 训练中 checkpoint（每 5000 步） |
| `./model/ppo_simple_final.zip` | 训练结束最终模型 |
| `./logs/single_training/` | SB3 csv、TensorBoard 事件、monitor.csv |
| `./logs/reward_components.csv` | 奖励分项逐步记录（comps 有值后才有内容） |

## 七、常见问题速查
