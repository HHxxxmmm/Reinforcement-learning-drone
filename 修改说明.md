# 修改说明

## 2026-06-25 接口文档对齐

本次修改根据课程接口文档校准奖励函数、初始化参数和单元测试，重点保证单位、血量尺度和攻击范围与平台约定一致。

### 奖励逻辑

- `utils/reward.py` 中血量尺度按接口文档统一为 `0-1000`，默认满血为 `1000`。
- 命中奖励按“一次命中扣 10 HP”计算，`enemy_damage` 使用 `20 * (掉血量 / 10)`，便于把每次有效命中作为明显正反馈。
- 新增 `attack_box` 奖励分项，用于刻画平台伤害触发区域：
  - 前向距离：`60-660m`
  - 横向误差：约 `±10m`
  - 机头越对准且目标越靠近攻击盒中心，奖励越高。
- 保留并继续记录以下奖励分项，方便 TensorBoard/CSV 分析：
  - `distance_progress`
  - `proximity`
  - `alignment`
  - `corridor`
  - `attack_box`
  - `enemy_damage`
  - `self_damage`
  - `survival`
  - `kill_bonus`
  - `death_penalty`
  - `total`
- 击落奖励和被击落惩罚分别调整为 `+300` / `-300`，强化回合胜负事件。

### 初始化逻辑

- `utils/initialize.py` 仍只返回 24 个 `int32`，即我方 12 项 + 敌方 12 项；`room_id`、`unit_id` 和其余包头由 `train_env.pack_initial()` 自动补齐。
- 我方初始位置保持为 `(0, 0, 30)`，即高度约 `300m`。
- 我方初始速度改为 `(30, 0, 0)`，按接口单位 `10m/s` 解释，约为 `300m/s`。
- 敌方初始位置保持随机前方 `120-180` 个位置单位，即约 `1200-1800m`，满足初始距离不少于 `1000m` 的作业要求。

### 截断逻辑

- `utils/truncate.py` 本次未改动。
- 当前函数接口只有 `my_state` 和 `enemy_state`，没有 episode 步数，因此不在这里实现超时截断。
- 回合超时仍交由平台的 `max_tick_per_round` 控制；如果后续需要 Python 侧主动超时，应在 `TrainEnv` 内维护 `step_count` 后再传入截断函数。

### 测试

- `tests/test_reward_initialize_truncate.py` 更新为接口文档尺度：
  - 默认 HP 为 `1000`
  - 单次命中扣 `10 HP`
  - 攻击盒前向范围为 `60-660m`
  - 横向偏离会降低 `attack_box` 奖励
  - 初始化速度按 `10m/s` 单位检查

验证命令：

```powershell
python -m unittest tests.test_reward_initialize_truncate
python -m compileall utils envs tests
```

## 2026-06-27 Simple 实测尺度修正

后续联调发现课程接口文档中的 HP 尺度与 Simple 场景实测不一致：文档写每方初始 HP 为 `1000`，但 Simple 观测包中 HP 为 `0-1` 归一化浮点，满血约为 `1.000`。因此本次修正以 Simple 实测为主，同时兼容旧文档尺度，避免后续切换场景时奖励完全失效。

### 奖励逻辑修正

- `utils/reward.py` 的默认 HP 改为 `1.0`。
- 掉血奖励自动识别 HP 尺度：
  - 当 HP 最大值不超过 `1.5` 时，按 Simple 归一化尺度处理，单次命中约等于 `0.01` HP。
  - 当 HP 最大值大于 `1.5` 时，按旧文档尺度处理，单次命中约等于 `10` HP。
- `enemy_damage` 和 `self_damage` 均按“命中次数”归一化后再乘权重，因此 `1.00 -> 0.99` 和 `1000 -> 990` 都会得到同等强度的奖励/惩罚。
- 固定靶机场景曾实测 `enemy_pos` 可能长期为 `(0,0,0)`。为了避免距离、视线角和攻击盒奖励把目标误判为原点，`reward.py` 在敌方位置全 0 时使用固定靶机 fallback 坐标 `(120, 0, 30)`，即前方约 `1200m`、高度约 `300m`。

### 初始化逻辑修正

- `utils/initialize.py` 将敌方初始位置从随机前方位置改为固定 `(120, 0, 30)`。
- 固定初始化更适合第一阶段 Simple vs 固定靶机先训通，也能和 reward 中的固定靶机 fallback 保持一致。
- 我方初始位置仍为 `(0, 0, 30)`，初速度仍为 `(30, 0, 0)`。

### 截断逻辑修正

- `utils/truncate.py` 也同步使用固定靶机 fallback。
- 当 `enemy_pos` 为 `(0,0,0)` 时，距离截断按固定靶机 `(120,0,30)` 计算，避免战机离原点较远但仍在靶机 5000m 范围内时被误判截断。

### 距离单位判断

- 目前没有发现与“位置单位 = 10m”相同强度的反证。
- 初始化、作业要求和已有测试仍按位置单位 `10m` 处理，因此 `POSITION_SCALE_M` 暂时保持 `10.0`。
- 如果后续 smoke test 明确显示观测位置已经是米制，再统一把距离比例和攻击盒阈值改为米制。

### 仍需提醒

- `utils/action.py` 当前仍然忽略 agent 输出并生成随机动作。这不属于本次 reward/initialize/truncate 修改范围，但会直接导致智能体无法真正学习，需要负责 action 的同学尽快修复。
- `utils/observation.py` 仍然返回全 0 观测，占位逻辑也会阻碍训练收敛，需要和观测设计同学联调。
